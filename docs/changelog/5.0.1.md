# Changelog v5.0.1

**Release Date:** 2026-01-14

## Summary

This release introduces the extension plugin system and enhances report
deduplication with multi-platform URL support.

## Changes

### Added

-   Extension plugin system with entry points support
-   Report deduplication extension plugin with multi-platform URL support
-   Enhanced logging with loguru for better debugging
-   Clickable source links for multi-platform news in HTML reports

### Changed

-   Updated default Ollama model to qwen2.5:14b-instruct for better
    deduplication accuracy
-   Normalized configuration keys to lowercase convention in extension plugins
-   Improved HTML report rendering to support multiple platform URLs

### Removed

-   Deprecated n8n workflow documentation (moved to separate repository)

### Fixed

-   Crawler execution mode configuration (RUN_MODE environment variable)
-   Report deduplication logic for better accuracy

## Technical Details

### Extension Plugin System

-   New plugin architecture with entry points
-   Base plugin class in `extensions/base.py`
-   Report deduplicate plugin in `extensions/report_dedupe/`

### Configuration Changes

```yaml
# Old format (uppercase)
report_dedupe:
  ENABLED: true
  STRATEGY: "ollama"
  SIMILARITY:
    THRESHOLD: 0.85

# New format (lowercase)
report_dedupe:
  enabled: true
  strategy: "ollama"
  similarity:
    threshold: 0.85
```

### HTML Report Enhancement

Multi-platform news now display as clickable links:

```
Source: 微博 / 知乎 / B站  (each links to respective platform)
```

## Migration Notes

1. Update extension config keys to lowercase
2. Update Ollama model if using custom configuration
3. No database changes required
